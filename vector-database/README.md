## What is a Vector Database?

A **vector database** is a specialized type of database designed to **store**, **index**, and **search** high-dimensional vector representations of data — known as **embeddings**. Unlike traditional databases that rely on exact matches vector databases use similarity search techniques such as cosine similarity or Euclidean distance to find items that are semantically or visually similar..

### Vector DB vs Traditional Database

| Traditional Database | Vector Database |
|---------------------|-----------------|
| Stores text, numbers, rows | Stores vectors (lists of numbers) |
| Finds **exact matches** (`WHERE name = 'John'`) | Finds **similar meaning** (closest vectors) |
| Good for structured data | Good for semantic / AI search |

Unlike traditional databases that rely on exact matches, vector databases use **similarity search** techniques such as:

- **Cosine similarity** — measures the angle between two vectors
- **Euclidean distance (L2)** — measures straight-line distance between two vectors

These help find items that are **semantically or visually similar** — even when the words are different.

```
Traditional DB:  "refund policy"  →  must match exactly
Vector DB:       "refund policy"  →  also finds "money back rules", "return policy"
```

## What are Embeddings?

**Embeddings** are dense numerical representations of data — such as words, sentences, images, or audio — mapped into a continuous high-dimensional space where **similar items are positioned closer together**.

### Key Points

- Generated by **machine learning models** that capture semantic meaning, context, and relationships
- Instead of comparing raw text or media directly, embeddings let systems measure **similarity through math**
- Enables faster search and extraction based on **meaning**, not exact text

### What Embeddings Enable

| Task | How Embeddings Help |
|------|---------------------|
| **Semantic search** | Find documents by meaning, not keywords |
| **Recommendation systems** | Suggest similar products, articles, or videos |
| **Clustering** | Group similar items together |
| **Classification** | Categorize text or images |
| **Cross-lingual matching** | Match content across different languages |

### Simple Example

```
"running shoes"     →  [0.12, -0.45, 0.88, 0.03, ...]
"jogging sneakers"  →  [0.11, -0.43, 0.90, 0.02, ...]  ← close in vector space!

"chocolate cake"    →  [0.91, 0.22, -0.55, 0.77, ...]  ← far away
```

Similar meaning → vectors are **close together**. Different meaning → vectors are **far apart**.

## How Do Embeddings Work?

Embeddings work by converting raw data (text, images, audio) into dense numerical vectors that **preserve meaning and relationships**.

### Step-by-Step

```
Raw Input (text, image, audio)
        │
        ▼
  Process through a model
  (Transformer for text, CNN for images)
        │
        ▼
  Extract key features
        │
        ▼
  Encode into fixed-length vector
  (high-dimensional space)
        │
        ▼
  Similar items → close together
  Different items → far apart
```

1. **Input** — raw data enters the model (e.g. a sentence or image)
2. **Feature extraction** — the model pulls out important patterns
3. **Encoding** — features become a fixed-length vector (e.g. 384 or 1536 numbers)
4. **Spatial arrangement** — similar items end up near each other in vector space
5. **Similarity measurement** — distance between vectors = how similar two items are

This allows search, recommendations, and classification to work based on **meaning** rather than exact matches.

## How Vector Databases Search

When you store embeddings in a vector database and run a query:

```
1. Your question → embedding model → query vector
2. Compare query vector to ALL stored vectors
3. Return the top K closest matches (nearest neighbors)
```

### Distance Metrics

| Metric | What It Measures |
|--------|------------------|
| **Cosine similarity** | Angle between vectors (good for text) |
| **Euclidean distance (L2)** | Straight-line distance (used in example below) |

## Simple Code Example (FAISS)

This example stores 3 sample vectors and performs a similarity search using **L2 (Euclidean) distance**. The `query_vector` is compared to all stored vectors, and the **top 2 most similar** vectors are returned.

```python
import faiss
import numpy as np

# 3 sample vectors (imagine these are embeddings of 3 documents)
data_vectors = np.array([
    [0.1, 0.2, 0.3, 0.4],
    [0.2, 0.1, 0.4, 0.3],
    [0.9, 0.8, 0.7, 0.6],
], dtype='float32')

dimension = data_vectors.shape[1]

# Create index and add vectors
index = faiss.IndexFlatL2(dimension)
index.add(data_vectors)

# Query: find vectors most similar to this one
query_vector = np.array([[0.1, 0.2, 0.3, 0.35]], dtype='float32')
distances, indices = index.search(query_vector, k=2)

print("Indices of closest vectors:", indices)
print("Distances from query:", distances)
```

### Output

```
Indices of closest vectors: [[0 1]]
Distances from query: [[0.0025 0.0325]]
```

### What This Means

- Vectors at index **0** and **1** are the closest to the query (smallest distance)
- Vector at index **2** `[0.9, 0.8, 0.7, 0.6]` is far away — not similar
- Lower distance = more similar

## Applications

### Image and Video Search

Finds visually similar media from a database.

- Feature embeddings are extracted from images or video frames
- Stored in the vector database
- A new query image retrieves the most visually similar results

### Question Answering Systems

Retrieves the most relevant information from large knowledge bases.

- Both queries and stored text are embedded into vectors
- The system compares vectors to find the closest match
- More accurate than simple keyword matching

### Cross-Lingual Information Retrieval

Supports matching across multiple languages using multilingual embeddings.

- Text in different languages is mapped into a **shared embedding space**
- Search in one language → retrieve relevant results in another

### Fraud and Anomaly Detection

Identifies unusual patterns by comparing embeddings with normal data.

- Store embeddings of typical behavior
- Detect when new data deviates significantly
- Helps flag fraudulent or suspicious activity early

## Vector DB in RAG

In a RAG pipeline, the vector database plays a critical role:

```
Documents → Chunk → Embed → Store in Vector DB
                                    │
User Question → Embed → Search Vector DB → Retrieve top chunks
                                    │
                          Send chunks + question to LLM → Answer
```

| RAG Step | Vector DB Role |
|----------|----------------|
| **Setup** | Store all document chunk embeddings |
| **Query time** | Find the most relevant chunks for the user's question |
| **Update** | Add new documents by embedding and storing them |

Without a vector database, RAG would have no fast way to find relevant context from thousands of documents.
